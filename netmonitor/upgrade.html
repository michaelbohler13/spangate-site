How to Update

The agent runs as a Docker container — updating takes two commands: pull the new image, restart the container. No reinstall, no reconfiguration. Your API key, device list, and settings stay exactly as they are.

bash — SSH into Pi, then run
cd ~/spangate-agent && docker compose pull && docker compose up -d

The container restarts automatically with the new image. Check the Dashboard → Agent card — the version number should update within 5 minutes once the new agent sends its first heartbeat.

bash — Terminal
cd ~/spangate-agent && docker compose pull && docker compose up -d

Same as Pi. The container restarts with the new image. Version updates in the Dashboard within 5 minutes.

powershell — run in your spangate-agent folder
cd "$env:USERPROFILE\spangate-agent"
docker compose pull
docker compose up -d

The container restarts with the new image. Version updates in the Dashboard within 5 minutes.

💡
That's it. No re-import, no new API key, no reconfiguration. All your device lists, groups, alert history, and config backups live in the cloud — they carry over automatically.

Frequently Asked Questions

Do I lose anything when I update?
No. Updating only replaces the Docker image. Your API key, config file, device list, alert history, and config backups are completely unaffected. The container restarts and reconnects automatically.
How long does the update take?
Usually under a minute. The docker compose pull downloads the new image (typically 100–200 MB), then docker compose up -d restarts the container in seconds. Monitoring resumes automatically.
Will devices show as DOWN during the update?
Briefly — while the container restarts (usually 10–30 seconds), no pings are sent. Devices won't immediately flip to DOWN since the backend allows a grace period before raising an alert. For most updates you'll see no alert noise at all.
Can I run both agents at the same time during migration?
Yes — briefly running both during a device migration is fine and even recommended. Both will send pings to the same backend. Stop the old agent once you've confirmed the new one is healthy.
I lost my API key — how do I find it?
Go to Dashboard → ⚙ Settings → Agent API Key. Your key is shown there and can be copied or regenerated. If you regenerate it, update config.yaml on your agent device before restarting.
