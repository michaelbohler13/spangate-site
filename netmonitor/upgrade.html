Upgrade & Migration
Guide

Moving to a bigger Pi, switching to Windows, or replacing a failed device? Select your platforms below and get step-by-step instructions.

Your API key is the key. All device lists, alert history, config backups, and groups are stored in the cloud. As long as the new device uses the same API key, everything carries over automatically — no re-import, no reconfiguration.

01
Export your device list (optional safety backup)
Saves a local CSV in case you want to verify or re-import.

Go to Dashboard → Devices and click ⬇ Export CSV. The file contains all your hostnames, IPs, vendors, groups, and SSH settings. SSH passwords are never exported for security.

💡
This step is optional. Your device list already lives in the cloud and will appear automatically on the new device. Export is just a local safety copy.
02
Install the agent on the new device
Follow the install guide for your new platform — use the same API key.

Follow the install guide for your new platform. When the wizard asks for your API Key, enter the exact same key you used on your old device. You can copy it from Settings → API Key.

Open Install Guide →
⚠️
Do not stop the old agent yet. Keep it running until you confirm the new device is online in Step 3.
03
Confirm the new agent is online
Watch the dashboard for the heartbeat to switch over.

Open the dashboard and check the Agent health card at the top of the page. Within 5 minutes you should see:

  • The Last seen timestamp updating
  • Device statuses refreshing
  • CPU / RAM / Temp metrics from the new device

Both old and new agents may run briefly at the same time — that's fine. You might see a few extra ping alerts during the overlap, but nothing is lost.

04
Stop the old agent
Once the new device is confirmed running, decommission the old one.
05
Re-enter SSH passwords (if needed)
Passwords are never stored in the cloud — a one-time update for SSH-enabled devices.

If any of your devices have SSH backup enabled, go to Dashboard → Devices, click ✎ Edit on each SSH-enabled device, and re-enter the password.

Everything else — device list, groups, ping monitoring, alert history, and config backups — is already there from your previous setup.

✓
You're done. All your data carried over automatically. The new device is now your active monitor.

Frequently Asked Questions

Can I run both agents at the same time?
Yes — briefly running both during migration is fine and even recommended. Both will send pings to the same backend. You may see a small number of duplicate "UP" alerts during the overlap window, but no data is lost and nothing breaks. Stop the old agent once you've confirmed the new one is healthy.
Will I lose my alert history or config backups?
No. Alert history and config backups are stored in the cloud against your site ID, not against any specific device. They are completely unaffected by changing the machine running the agent.
Do I need to re-add all my devices on the new device?
No. Your device list lives in the cloud. The new agent fetches it automatically within 5 minutes of starting up. You only need to re-enter SSH passwords for devices that have SSH backup enabled, since passwords are never stored in the cloud.
Can I change my site name when migrating?
Yes — just enter a different site name in the setup wizard on the new device. The site name is just a display label. Your data is linked to your API key, not the site name.
What if my new device can't reach some devices on the network?
The monitoring agent needs network access (same VLAN or routed path) to the devices it monitors. If the new device is on a different network segment, some devices may show as DOWN. Check routing between the new host and your managed devices. Running ping 10.x.x.x from the new device is a quick test.
I lost my API key — how do I find it?
Go to Network Monitor → Settings. Your API key is shown there and can be copied or regenerated. If you regenerate it, update the key on both old and new agents before stopping the old one.
