Windows
Install Guide

Run the Network Monitor agent on Windows 10 or 11 as a background service using NSSM. Survives reboots — no logged-in user required.

🍓 Raspberry Pi ⊞ Windows
Requirements
  • Windows 10 (version 1903 or later) or Windows 11
  • Python 3.9 or higher
  • Network access to your managed devices — same VLAN or routed
  • SSH enabled on all target switches and routers
  • Your SpanGate API key — from Network Monitor → Settings
  • Git for Windows — optional but recommended
01
Install Python
Python 3.9 or higher is required. The installer must add Python to your PATH.
  1. Go to python.org/downloads
  2. Click Download Python 3.11.x (or any 3.9+ release)
  3. Run the installer
  4. ✅ Check "Add Python to PATH" on the first screen — do not skip this
  5. Click Install Now

Verify in Command Prompt:

cmd
python --version
pip --version
⚠
Both commands must succeed. If python is not recognized, re-run the installer and ensure "Add to PATH" is checked.
02
Install Git (Recommended)
Git makes downloading and updating the agent much easier.
  1. Go to git-scm.com/download/win
  2. Download and run the installer
  3. All default settings are fine — click Next through each screen
cmd
git --version
03
Download the Agent
Run one command in PowerShell to download the agent files.
powershell
irm https://spangate-site.vercel.app/netmonitor/agent-install.ps1 | iex
cd "$env:USERPROFILE\Documents\spangate-agent\netmonitor-agent"
✓
The script downloads only the agent files to Documents\spangate-agent\netmonitor-agent. To update later, re-run the same irm ... | iex command.
04
Create Virtual Environment & Install Dependencies
Isolate the agent's packages from your system Python.
cmd
python -m venv venv
venv\Scripts\activate
pip install --upgrade pip
pip install -r requirements.txt
✓
You should see (venv) at the start of your prompt when the virtual environment is active. Installation takes 2–5 minutes.
05
Configure the Agent
Set your API key, site name, and add your devices.
  1. In File Explorer, navigate to Documents\spangate-agent\netmonitor-agent
  2. Copy config.yaml and rename the copy to my-config.yaml
  3. Open my-config.yaml in Notepad or VS Code
  4. Edit the following fields:
yaml
agent:
  site_name: "Main Office"             # ← your site name
  api_url:   "https://spangate-site-r81b.vercel.app"
  api_key:   "spng_paste_your_key_here"  # ← your SpanGate API key
  ping_interval:        60
  config_pull_interval: 604800

devices:
  - hostname:     core-sw-01
    ip:          10.1.1.1              # ← your device IP
    vendor:      cisco
    device_type: cisco_ios
    ssh_username: admin
    ssh_password: "your-switch-password"   # ← SSH credentials
    ssh_port:    22

Add one entry under devices: for each switch or router. Save the file.

06
Test the Agent Manually
Run once to confirm it connects to the API and reaches your devices.

Make sure the virtual environment is active ((venv) in prompt), then:

cmd
python agent.py --config my-config.yaml

Expected output:

2026-05-25 14:32:01 INFO SpanGate Network Monitor Agent v1.0.0 — Main Office — 3 device(s)
2026-05-25 14:32:01 INFO [PING] Loop started — 3 device(s), interval 60s
2026-05-25 14:32:01 INFO [SSH] Pull loop started — interval 604800s (168.0 hours)
2026-05-25 14:32:01 INFO [HB] Heartbeat loop started — interval 300s
2026-05-25 14:32:02 INFO [PING] core-sw-01 (10.1.1.1) initial state: up

Stop with Ctrl+C once confirmed working.

07
Run as a Windows Service with NSSM
NSSM keeps the agent alive across reboots without needing a logged-in user.

Install NSSM:

  1. Go to nssm.cc/download
  2. Download the latest release ZIP
  3. Extract it and copy nssm.exe from the win64 folder to C:\Windows\System32\

Install the service — open Command Prompt as Administrator:

cmd (admin)
nssm install SpangateNetMonitor

Fill in the NSSM GUI as follows — use your actual username (check with echo %USERNAME%):

FieldValue
Tab Application
Path C:\Users\YourName\Documents\spangate-agent\netmonitor-agent\venv\Scripts\python.exe
Startup directory C:\Users\YourName\Documents\spangate-agent\netmonitor-agent
Arguments agent.py --config my-config.yaml
Tab Details
Display name SpanGate Network Monitor
Description SpanGate Technology — Network Monitor Agent
⚠
Always use absolute paths in NSSM. The Path must point to venv\Scripts\python.exe — not the system Python.

Click Install service, then start it:

cmd (admin)
nssm start SpangateNetMonitor
08
Verify the Service
Confirm the agent is running and reporting to your dashboard.

Open Services — press Win+R, type services.msc, press Enter. Find SpanGate Network Monitor — status should be Running.

●
Your SpanGate dashboard should show ● Agent reporting within a few minutes of the service starting.

Updating the Agent

cmd
irm https://spangate-site.vercel.app/netmonitor/agent-install.ps1 | iex
nssm restart SpangateNetMonitor

Windows Firewall Note

The agent uses ICMP (ping) for device status checks. Windows Firewall may block outbound ICMP on some configurations. If devices show as DOWN even though they're reachable:

  1. Open Windows Defender Firewall with Advanced Security
  2. Go to Outbound Rules → New Rule
  3. Rule type: Custom
  4. Protocol: ICMPv4
  5. Action: Allow the connection
  6. Apply to all profiles, name it SpanGate ICMP Out, click Finish

Troubleshooting

ProblemFix
python not recognized Re-run the installer and check "Add Python to PATH"
pip install fails on netmiko Run pip install --upgrade pip first, then retry
ICMP ping shows all devices DOWN Allow outbound ICMPv4 in Windows Defender Firewall (see above)
SSH connection refused to device Verify SSH is enabled on the switch and port 22 is open
Service won't start Use absolute paths in NSSM — not relative paths
Agent runs manually but not as service The Path must point to venv\Scripts\python.exe, not the system Python
Config pull fails Ensure the Windows machine has a route to the device management IP
API key rejected (401) Copy the key from the dashboard exactly — no extra spaces
nssm not found Verify nssm.exe is in C:\Windows\System32\ and re-open Command Prompt
