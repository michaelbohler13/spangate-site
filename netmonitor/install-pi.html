Raspberry Pi
Install Guide

Get the Network Monitor agent running on a Raspberry Pi in under 15 minutes. A Pi 4 (4 GB) handles up to 200 devices.

🍓 Raspberry Pi ⊞ Windows ⌘ macOS
⚠️
Authorized Use Only. Only deploy the SpanGate agent on networks and devices you own or have received explicit written authorization to monitor. Unauthorized monitoring may violate the Computer Fraud and Abuse Act (18 U.S.C. § 1030) and other state and federal laws. By proceeding you confirm you have proper authorization. See Terms of Service — Section 2.
Requirements
  • Raspberry Pi — Zero 2W, Pi 4, or Pi 5 (Pi 4 4 GB recommended)
  • Raspberry Pi OS Lite, 64-bit (Bullseye or Bookworm)
  • Python 3.9 or higher — pre-installed on Bullseye and later
  • Network access to your managed devices — same VLAN or routed
  • SSH enabled on all target switches and routers
  • Your SpanGate API key — from Network Monitor → Settings

01
Flash Raspberry Pi OS
Use Raspberry Pi Imager to write OS Lite to your microSD card with SSH pre-enabled.
  1. Download Raspberry Pi Imager from raspberrypi.com/software
  2. Insert a microSD card — 8 GB minimum, 16 GB recommended
  3. Choose Device → select your Pi model
  4. Choose OS → Raspberry Pi OS Lite (64-bit)
  5. Choose Storage → select your microSD card
  6. Click Next — when prompted "Would you like to apply OS customisation settings?" click Edit Settings
⚙️
In the General tab: set hostname to spangate-monitor, username to spangate, choose a strong password, and configure Wi-Fi if not using Ethernet.
In the Services tab: enable SSH and select Use password authentication.
In the Remote Access tab: check Enable Raspberry Pi Connect — this lets you access a terminal from any browser without needing an IP address (recommended).
Click Save, then Yes to apply.
  1. Confirm erasing the card when prompted → Imager will write and verify
  2. Insert the microSD into your Pi and power it on
02
Connect to Your Pi
Give the Pi 60 seconds to boot. Use Raspberry Pi Connect for the easiest experience, or go old school with SSH.

Option A — Raspberry Pi Connect RECOMMENDED
Browser-based terminal — no SSH client, no IP address, works from anywhere.

  1. If you checked Enable Raspberry Pi Connect in Imager (Step 1), your Pi is already set up. If not, SSH in first and run:
    bash
    sudo apt install -y rpi-connect && rpi-connect signin
  2. Visit connect.raspberrypi.com — sign in with your free Raspberry Pi account (or create one)
  3. Your Pi will appear as spangate-monitor — click it then choose Remote Shell
  4. A terminal opens in your browser — you're in. Skip to Step 3 →

Prefer SSH? Pick one of the options below:

Option B — Try hostname first (works on most home networks):

bash
ssh spangate@spangate-monitor.local

Option C — Find the IP via your router (most reliable):
Log into your router admin page (usually 192.168.1.1 or 192.168.0.1) and look under DHCP Clients or Connected Devices for spangate-monitor. Then SSH using the IP shown:

bash
ssh spangate@192.168.1.XXX

Option D — Scan for the Pi on Windows:
Open Command Prompt and run:

cmd
arp -a

Look for a device with hostname spangate-monitor or a Raspberry Pi MAC address starting with b8-27-eb, dc-a6-32, or e4-5f-01. Or download Advanced IP Scanner (free) — it lists every device on your network.

💡
First time connecting via SSH? Your terminal will ask "Are you sure you want to continue connecting?" — type yes and press Enter. This is normal and only happens once.
03
Install Docker & Run the Setup Wizard
One command installs Docker, pulls the agent image, and starts it automatically. Your API key is pre-filled — just copy and paste.

Fill in your API key and site name below, then click Copy and paste into your Pi terminal. That's it.

⚡ Quick Setup Wizard
Copy from Settings → API Key
Shown in your dashboard (e.g. Main Campus)
bash — ready to paste into your Pi terminal
export SPNG_API_KEY="paste key above" export SPNG_SITE_NAME="enter site name above" curl -fsSL https://spangate-site-r81b.vercel.app/netmonitor-agent/setup.sh | bash
✓
The wizard installs Docker automatically, asks for your site name, pulls the agent image, and starts it as a container that restarts on reboot. The whole process takes about 3 minutes.

Expected output when complete:

[OK] Docker is ready.
[OK] API key accepted.
[INFO] Pulling latest agent image from Docker Hub...
[INFO] Starting SpanGate agent...
[OK] Agent is running — you're all set!

Useful Commands

bash
docker logs -f spangate-agent          # view live agent logs
cd ~/spangate-agent && docker compose down    # stop the agent
cd ~/spangate-agent && docker compose up -d  # start the agent
docker pull spangate/netmonitor-agent:latest  # update to latest version

Troubleshooting

ProblemFix
Docker not found after install Log out and log back in — the Pi needs to reload your user group
Container starts then exits Run docker logs spangate-agent to read the error
API key rejected (401) Copy the key from the dashboard exactly — no extra spaces
Device shows DOWN but is reachable Confirm the Pi has a route: run ping 10.1.1.1 from the Pi
SSH config pull fails for Aruba CX Run web-management ssl on the switch first
.local hostname not resolving Install avahi-daemon: sudo apt install avahi-daemon
