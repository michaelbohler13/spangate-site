Raspberry Pi
Install Guide

Get the Network Monitor agent running on a Raspberry Pi in under 15 minutes. A Pi 4 (4 GB) handles up to 150 devices.

🍓 Raspberry Pi ⊞ Windows
Requirements
  • Raspberry Pi — Zero 2W, Pi 4, or Pi 5 (Pi 4 4 GB recommended)
  • Raspberry Pi OS Lite, 64-bit (Bullseye or Bookworm)
  • Python 3.9 or higher — pre-installed on Bullseye and later
  • Network access to your managed devices — same VLAN or routed
  • SSH enabled on all target switches and routers
  • Your SpanGate API key — from Network Monitor → Settings

Recommended Hardware

Pi Model Max Devices Approx. Cost
Pi Zero 2W 50 ~$50
Pi 4 (4 GB) Recommended 200 ~$150
Pi 5 (8 GB) 1,000 ~$249
01
Flash Raspberry Pi OS
Use Raspberry Pi Imager to write OS Lite to your microSD card with SSH pre-enabled.
  1. Download Raspberry Pi Imager from raspberrypi.com/software
  2. Insert a microSD card — 8 GB minimum, 16 GB recommended
  3. Choose Device → select your Pi model
  4. Choose OS → Raspberry Pi OS Lite (64-bit)
  5. Choose Storage → select your microSD card
  6. Click the gear icon (⚙️) before clicking Write to pre-configure:
⚙️
In the gear/advanced options: enable SSH, set hostname to spangate-monitor, set username to spangate, choose a strong password, and configure Wi-Fi if not using Ethernet.
  1. Click Write and wait for it to finish
  2. Insert the microSD into your Pi and power it on
02
Connect to Your Pi
Give the Pi 60 seconds to boot, then SSH in.
bash
ping spangate-monitor.local
ssh spangate@spangate-monitor.local

If .local doesn't resolve, check your router's DHCP table for the Pi's IP and SSH directly:

bash
ssh spangate@192.168.1.XXX
03
Update the System
Bring the OS fully up to date and install required packages.
bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y python3 python3-pip python3-venv git
python3 --version
✓
The version must be 3.9 or higher. Raspberry Pi OS Bullseye and Bookworm both ship with Python 3.11.
04
Download the Agent
Clone the SpanGate repo to your Pi's home directory.
bash
cd ~
git clone https://github.com/michaelbohler13/spangate-site.git
cd spangate-site/netmonitor-agent
05
Create Virtual Environment & Install Dependencies
Isolate the agent's Python packages in a venv.
bash
python3 -m venv venv
source venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
⏱
Installation takes 2–5 minutes on a Pi 4. The netmiko and napalm packages are the heaviest.
06
Configure the Agent
Set your API key, site name, and add your devices.
bash
sudo mkdir -p /etc/spangate/netmonitor
sudo cp config.yaml /etc/spangate/netmonitor/config.yaml
sudo chmod 600 /etc/spangate/netmonitor/config.yaml
sudo nano /etc/spangate/netmonitor/config.yaml

Edit the following fields — add one entry under devices: for each switch or router:

yaml
agent:
  site_name: "Main Campus"          # ← your site name
  api_url:   "https://api.spangate.com"
  api_key:   "spng_paste_your_key_here" # ← your SpanGate API key
  ping_interval:        60
  config_pull_interval: 604800

devices:
  - hostname:     core-sw-01
    ip:          10.1.1.1             # ← your device IP
    vendor:      cisco
    device_type: cisco_ios
    ssh_username: admin
    ssh_password: "your-switch-password"  # ← SSH credentials
    ssh_port:    22

Save with Ctrl+X → Y → Enter. Your API key is in the dashboard under Settings.

07
Test the Agent Manually
Run once to confirm it connects to the API and reaches your devices.
bash
cd ~/spangate-site/netmonitor-agent
source venv/bin/activate
python3 agent.py --config /etc/spangate/netmonitor/config.yaml

Expected output:

2026-05-25 14:32:01 INFO SpanGate Network Monitor Agent v1.0.0 — Main Campus — 3 device(s)
2026-05-25 14:32:01 INFO [PING] Loop started — 3 device(s), interval 60s
2026-05-25 14:32:01 INFO [SSH] Pull loop started — interval 604800s (168.0 hours)
2026-05-25 14:32:01 INFO [HB] Heartbeat loop started — interval 300s
2026-05-25 14:32:02 INFO [PING] core-sw-01 (10.1.1.1) initial state: up

Stop with Ctrl+C once confirmed working.

08
Install as a Systemd Service
Run the agent automatically on boot — no logged-in user required.
bash
sudo nano /etc/systemd/system/spangate-netmonitor.service

Paste the following (adjust username if you used something other than spangate):

systemd unit
[Unit]
Description=SpanGate Network Monitor Agent
Documentation=https://spangate.com/netmonitor
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=spangate
WorkingDirectory=/home/spangate/spangate-site/netmonitor-agent
ExecStart=/home/spangate/spangate-site/netmonitor-agent/venv/bin/python3 agent.py --config /etc/spangate/netmonitor/config.yaml
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal
SyslogIdentifier=spangate-netmonitor

[Install]
WantedBy=multi-user.target

Save with Ctrl+X → Y → Enter, then enable and start:

bash
sudo systemctl daemon-reload
sudo systemctl enable spangate-netmonitor
sudo systemctl start spangate-netmonitor
09
Verify the Service
Confirm the agent is running and reporting to your dashboard.
bash
sudo systemctl status spangate-netmonitor

You should see Active: active (running). Follow live logs:

bash
sudo journalctl -u spangate-netmonitor -f
●
Your SpanGate dashboard should show ● Agent reporting within a few minutes of the service starting.

Updating the Agent

bash
cd ~/spangate-site
git pull
sudo systemctl restart spangate-netmonitor

Troubleshooting

ProblemFix
python3 shows 3.8 or lower Run sudo apt install python3.11 python3.11-venv then retry
SSH connection refused to device Verify SSH is enabled on the switch and port 22 is open
Agent starts then immediately stops Run sudo journalctl -u spangate-netmonitor -n 50 to read the error
API key rejected (401) Copy the key from the dashboard exactly — no extra spaces
Device shows DOWN but is reachable Confirm the Pi has a route: run ping 10.1.1.1 from the Pi
Config pull fails for Aruba CX Run web-management ssl on the switch first
netmiko install fails Run sudo apt install python3-dev libssl-dev then retry pip install
Service says failed status Check permissions: ls -la /etc/spangate/netmonitor/config.yaml — should be -rw-------
.local hostname not resolving Install avahi-daemon on the Pi: sudo apt install avahi-daemon
