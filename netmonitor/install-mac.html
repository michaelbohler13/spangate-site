macOS
Install Guide

Run the Network Monitor agent on a Mac using Docker Desktop. Works on both Intel and Apple Silicon. A setup wizard handles everything in under 5 minutes.

🍓 Raspberry Pi ⊞ Windows ⌘ macOS
⚠️
Authorized Use Only. Only deploy the SpanGate agent on networks and devices you own or have received explicit written authorization to monitor. Unauthorized monitoring may violate the Computer Fraud and Abuse Act (18 U.S.C. § 1030) and other state and federal laws. By proceeding you confirm you have proper authorization. See Terms of Service — Section 2.
Requirements
  • macOS 12 Monterey or later (Intel or Apple Silicon)
  • Docker Desktop — free, download here
  • Network access to your managed devices — same VLAN or routed
  • Your SpanGate API key — from Network Monitor → Settings
01
Install Docker Desktop
Docker Desktop runs the agent in a container — no Python or Homebrew needed.
  1. Go to docker.com/products/docker-desktop — choose Mac with Apple Chip or Mac with Intel Chip as appropriate
  2. Open the downloaded .dmg and drag Docker to your Applications folder
  3. Launch Docker from Applications and wait for the whale icon in the menu bar to go solid (≈ 30 seconds)
ℹ
macOS may ask you to allow Docker to install a helper tool — click Install Helper and enter your Mac password when prompted.
02
Run the Setup Wizard
Open Terminal and paste two lines — the wizard handles everything else.

Fill in your API key and site name below, then click Copy and paste into Terminal (⌘ Space, type Terminal, press Enter).

⚡ Quick Setup Wizard
Copy from Settings → API Key
Shown in your dashboard (e.g. Main Campus)
bash — ready to paste into Terminal
export SPNG_API_KEY="paste key above" export SPNG_SITE_NAME="enter site name above" curl -fsSL https://spangate-site.vercel.app/netmonitor-agent/setup.sh | bash
✓
The wizard checks for Docker, pulls the agent image, and starts it as a container that restarts automatically whenever Docker Desktop starts.

Expected output when complete:

[OK] Docker is ready.
[OK] API key accepted.
[INFO] Pulling latest agent image from Docker Hub...
[INFO] Starting SpanGate agent...
[OK] Agent is running — you're all set!

The agent container is named spangate-agent and restarts automatically whenever Docker Desktop starts.

Useful Commands

bash
docker logs -f spangate-agent                        # view live agent logs
cd ~/spangate-agent && docker compose down            # stop the agent
cd ~/spangate-agent && docker compose up -d           # start the agent
docker pull spangate/netmonitor-agent:latest          # update to latest version

Troubleshooting

ProblemFix
Docker Desktop not running Launch Docker from your Applications folder and wait for the menu bar whale icon to go solid before re-running the script
API key rejected (401) Copy the key from the dashboard exactly — no extra spaces. Key must start with spng_
Device shows DOWN but is reachable Docker Desktop on Mac routes traffic through its Linux VM — confirm devices are reachable from this Mac: ping 10.1.1.1
Container starts then stops Run docker logs spangate-agent to read the error message
curl: command not found curl is built into macOS — if missing, install Xcode Command Line Tools: xcode-select --install
Agent stopped after reboot Enable Docker Desktop to start on login: Docker menu → Settings → General → "Start Docker Desktop when you sign in"
